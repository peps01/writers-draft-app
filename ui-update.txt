WRITER'S DRAFT APP - COMPLETE UI DOCUMENTATION
================================================================================

AUTHOR: Peps (pepepmjegano@gmail.com)
VERSION: 0.0.1
STACK: Docker-based monorepo — Django REST backend + Quasar/Vue 3 frontend + PostgreSQL

PURPOSE: A full-stack novel-writing web application. Writers can manage projects,
write scenes in a rich text editor, maintain a Story Bible (characters, places,
timeline events), track writing statistics, export to EPUB, and get AI-powered
writing assistance via Google Gemini.

FRONTEND FRAMEWORKS/LIBRARIES:
- Vue 3 (Composition API, <script setup>)
- Quasar 2 (q-btn, q-input, q-dialog, q-card, q-select, q-icon, q-layout, q-header,
  q-drawer, q-tabs, q-tab-panels, q-form, q-banner, q-skeleton, q-spinner, q-btn-toggle,
  q-checkbox, q-toggle, etc.)
- Vue Router 5 (hash mode routing)
- Pinia 2 (state management)
- Axios (HTTP client)
- TipTap 3 / ProseMirror (rich text editor)
- Chart.js 4 + vue-chartjs (statistics charts)
- SortableJS (drag-and-drop reordering)
- Google Fonts: Playfair Display, Inter, Lora

BACKEND: Django 5 + Django REST Framework + PostgreSQL 16
EXTERNAL: Google Gemini API (gemini-2.5-flash), EPUB export via EbookLib

2. ROUTER / ROUTES
================================================================================

File: /frontend/src/router/routes.js
File: /frontend/src/router/index.js

Route mode: hash mode (createWebHashHistory)

Route table:
  /                  -> redirects to /dashboard
  /login             -> LoginPage.vue        [guest only]
  /register          -> RegisterPage.vue     [guest only]
  /dashboard         -> DashboardPage.vue    [auth required]
  /settings          -> SettingsPage.vue     [auth required]
  /projects/:id      -> ProjectDetailPage.vue [auth required]
  /projects/:id/story-bible  -> StoryBiblePage.vue [auth required]
  /projects/:id/write        -> WritePage.vue      [auth required]
  /projects/:id/statistics   -> StatisticsPage.vue [auth required]
  /:catchAll(.*)*    -> ErrorNotFound.vue    [no guard]

Route guards (in router/index.js):
  - requiresAuth meta: redirects unauthenticated users to /login
  - guest meta: redirects authenticated users to /dashboard
  - beforeEach guard fetches CSRF token if not present

9. ARCHITECTURAL NOTES
================================================================================

Authentication: Django session-based (sessionid cookie), CSRF via csrftoken cookie.
Axios reads csrftoken cookie and sends as X-CSRFToken header on mutating requests.

Dark mode: Persisted to localStorage('wda_dark_mode'), restored on boot
(boot/darkmode.js). App.vue watches $q.dark changes.

Editor auto-save: Scene content auto-saves 2.5s after last edit.
Scene tags auto-save on change.

Auto-tagging: Editor scans scene content every 3s and auto-selects
character/place/timeline tags matching entity names found in content.

Version snapshots: Every PATCH to scene content creates a SceneVersion record.
Restore requires user's password.

AI Assistant: Conversations scoped per-scene. System prompt includes tagged
Story Bible entities + current scene content. Free-tier warning shown once
per session (sessionStorage flag).

View mode persistence: Story Bible grid/list preference stored in localStorage.

Font settings: Font family + size stored in localStorage, restored on editor mount.

Fullscreen mode: WritePage toggles fullscreen, hiding the scene board panel.

Layout approach: Each page defines its own <q-layout> — no shared layout wrapper
(original MainLayout.vue is unused).

EPUB export: Generates .epub via backend EbookLib, returns as downloadable file.
User selects which scenes to include.
